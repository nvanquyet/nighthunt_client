## 🎯 Key Concepts

### Three Main System Areas

1. **Weapon System** (Shooting logic)
   - ProjectileWeapon: Fires projectiles
   - ProjectileSpawner: Network synchronization
   - ProjectileComponent: Projectile behavior

2. **UI System** (Display logic)
   - GameHUD: Main orchestrator
   - CombatHUDPanel: Weapon & quick slots display
   - WeaponSlotButton: Individual weapon slot (×3)
   - QuickSlotHUDButton: Individual quick slot (×4)

3. **Player System** (Stat management)
   - PlayerStatSystem: Health, stamina, armor
   - InventorySystem: Item management
   - WeaponSystem: Weapon equip/unequip
   - (Other systems)

All three communicate via **events** (no polling!)

---

## 📡 Event-Driven Architecture

Key events you'll see:

```
WeaponSystem.OnAmmoChanged
  └─→ CombatHUDPanel listens
      └─→ Updates all weapon slot buttons
          └─→ WeaponSlotButton.RefreshAmmo()
              └─→ _ammoText.text = "30 / 90"

WeaponSystem.OnActiveWeaponChanged
  └─→ CombatHUDPanel listens
      └─→ Updates selected border on active slot
          └─→ All buttons refresh selected state

WeaponSystem.OnReloadStateChanged
  └─→ CombatHUDPanel listens
      └─→ Shows/hides reload indicator
          └─→ WeaponSlotButton shows cooldown ring
```

**No Polling!** Everything reactive.

## 💡 Pro Tips

1. **Use Setup Tool First**
   - It does 80% of work automatically
   - Save time on manual configuration

2. **Follow Exact Order**
   - Configs → Prefabs → Player → UI
   - Other order causes dependency issues

3. **Keep Console Visible**
   - Watch for debug.Log messages
   - They tell you if setup complete

4. **Test After Each Phase**
   - Don't wait until end to test
   - Catch issues early

5. **Reference Documents**
   - Have Visual Reference open while setting up
   - Compare your setup to diagrams

6. **Use Layers**
   - Create "Projectile" layer
   - Prevents projectiles from hitting player who fired

7. **Name Things Clearly**
   - "PrimaryWeaponSlot" not "Button1"
   - Makes debugging easier later

## 🐛 Debugging Workflow

### If something doesn't work:

1. **Check Console** (Ctrl+Shift+C)
   - Look for red errors
   - Read error messages carefully

2. **Verify Components**
   - Use Visual Reference diagram
   - Compare your hierarchy to it

3. **Check Inspector**
   - Are all fields assigned?
   - Use Setup Guide for correct values

4. **Test Incrementally**
   - Don't add everything at once
   - Test each phase before moving on

5. **Read Error Message**
   - Usually tells you exactly what's wrong
   - Search the document for that keyword

## 📞 FAQ (Frequently Asked Questions)

**Q: Can I use Hitscan weapons instead of Projectile?**
A: Yes! Check [WEAPON_TYPES_ADVANCED.md](WEAPON_TYPES_ADVANCED.md) for HitscanWeapon setup.

**Q: Do I need to use the Setup Tools?**
A: No, but they save tons of time. Manual setup is in Guide Section 5.

**Q: Can I have both ProjectileWeapon and HitscanWeapon on same player?**
A: Yes! Primary slot can be Hitscan, Secondary can be Projectile.

**Q: How do I add more weapon slots?**
A: See [GAMEPLAY_SETUP_GUIDE.md - Section 2.2](GAMEPLAY_SETUP_GUIDE.md#22-weaponslotbutton-setup-chi-tiết)

**Q: What about network synchronization?**
A: ProjectileSpawner handles it with ServerRpc/ObserversRpc. Details in [WEAPON_TYPES_ADVANCED.md](WEAPON_TYPES_ADVANCED.md#network-optimization)

**Q: Can I customize the UI colors/layout?**
A: Yes! UI is pure Unity UI, fully customizable. Just keep component references linked.

**Q: How do I debug why projectile isn't spawning?**
A: See Quick Card - "Projectile not visible?" section or Guide - Emergency Fixes
