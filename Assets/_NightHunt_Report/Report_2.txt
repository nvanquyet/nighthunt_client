1. HỆ THỐNG FOG OF WAR (FOW) — PHÂN TÍCH ĐẦY ĐỦ

Kiến trúc hiện tại:
FOW Package (Asset: FogOfWar/)
  FogOfWarRevealer3D  — tạo "vùng nhìn thấy" quanh player
  FogOfWarHider       — ẩn object khi nằm ngoài vùng nhìn thấy
  HiderDisableRenderers — tắt Renderer khi bị ẩn

Custom Binders:
  FogVisionBinder         — bind VisionRange stat → Revealer.ViewRadius
  FogTeamVisibilityBinder — quyết định object có bị ẩn không (dựa theo team)
  FogTeamDebugController  — debug tool (F9/F10 swap teams)


Flow hoạt động:
1. Player spawn → FogTeamVisibilityBinder.Start()
   → TryGetLocalTeamId() từ SpectateManager.GetLocalPlayer().TeamId
   → Nếu object là enemy → EnsureHiderExists() → thêm FogOfWarHider + HiderDisableRenderers
   → Nếu object là ally → RemoveHiderIfExists()

2. FogVisionBinder.OnEnable()
   → Đọc PlayerStatType.VisionRange từ IPlayerStatSystem
   → Set FogOfWarRevealer3D.ViewRadius = visionRange
   → Subscribe OnStatChanged để update khi stat thay đổi

3. Khi player chết:
   → CharacterLifecycleController.OnDied → FogVisionBinder.RefreshRevealerState()
   → _revealer.enabled = false (không reveal FOW khi chết)

4. Khi team thay đổi (SyncVar):
   → NetworkPlayer.OnPublicDataChanged → FogTeamVisibilityBinder.RefreshVisibilityForLocalTeam()
   → Re-evaluate tất cả objects


⚠ VẤN ĐỀ FOW:

FOW-1: Spectator mode không update FOW đúng
Khi spectate player khác:
  - SpectateManager.GetLocalPlayer() vẫn trả về LOCAL player
  - FogTeamVisibilityBinder dùng LOCAL team để quyết định visibility
  - → Spectator thấy FOW của mình, KHÔNG phải của người đang spectate
  
FIX: Khi spectate bắt đầu:
  1. Disable local player's FogOfWarRevealer3D
  2. Enable spectated player's FogOfWarRevealer3D (nếu cùng team)
     HOẶC disable FOW hoàn toàn (nếu khác team — spectator thấy tất cả)
  3. Khi spectate kết thúc → restore


FOW-2: Late-join race condition (đã có workaround nhưng fragile)
WaitForNetworkOwner() retry 60 frames → OK nhưng:
  - Nếu NetworkPlayer.TeamId SyncVar đến sau frame 60 → visibility sai
  - SubscribeLocalPlayerTeamChange() đã handle case này → OK
  - Nhưng nếu local player chưa spawn khi remote player's binder chạy
    → default visible (đúng) nhưng không có guarantee re-check sau
  
FIX: Đã có OnLocalPlayerSet event → OK. Cần verify test case late-join.


FOW-3: Deployables (VisionWard, RespawnBeacon) không có team info rõ ràng
BaseDeployable.OwnerTeamId được dùng nhưng:
  - Không rõ OwnerTeamId được set khi nào (spawn? equip?)
  - Nếu set sau khi FogTeamVisibilityBinder.Start() → visibility sai
  
FIX: BaseDeployable.OwnerTeamId phải được set TRƯỚC khi NetworkObject spawn
     hoặc dùng SyncVar để trigger RefreshVisibilityForLocalTeam()

2. HỆ THỐNG CAMERA — PHÂN TÍCH ĐẦY ĐỦ

Kiến trúc hiện tại:
CameraStateManager (per-player, trên PlayerPrefab)
  States: Free | Locked | WeaponAim
  - Free: CinemachineInputAxisController enabled (mouse rotate)
  - Locked: InputAxisController disabled (frozen rotation)
  - WeaponAim: InputAxisController disabled (character facing drives aim)
  
GameCameraController (scene-level, trong 02_Map_01)
  - Singleton (_instance)
  - Follow SpectateManager.OnCurrentPlayerChanged
  - Set CinemachineCamera.Follow + LookAt = target.transform
  - Auto-spectate khi local player chết
  
SpectatorInputHandler
  - NextPlayer / PreviousPlayer / FreeCamera (exit spectate)
  - Cooldown 0.5s giữa các lần switch


Flow Camera khi Spectate:
1. Local player chết → CharacterLifecycleController.OnDied
2. GameCameraController.HandleLocalPlayerDied()
   → TryStartSpectatingNextAlive()
   → SpectateManager.StartSpectating(firstCandidate)
3. SpectateManager.OnCurrentPlayerChanged → GameCameraController.HandleCurrentPlayerChanged()
   → SetCameraTarget(player.transform)
   → CinemachineCamera.Follow = target.transform
   → CinemachineCamera.LookAt = target.transform
4. CharacterInputLifecycle.HandleSpectateStarted()
   → InputLayerManager.TransitionToState(InputState.Spectating)
   → Spectator ActionMap enabled, Player/Combat disabled
5. SpectatorInputHandler: Tab/Q/E → SwitchSpectatedPlayer()


⚠ VẤN ĐỀ CAMERA:

CAM-1: CameraStateManager và GameCameraController không liên kết
CameraStateManager (per-player) quản lý Free/Locked/WeaponAim
GameCameraController (scene) quản lý Follow target

Khi spectate:
  - GameCameraController đổi Follow target → OK
  - Nhưng CameraStateManager vẫn ở state cũ (WeaponAim/Locked)
  - → Camera bị frozen khi spectate nếu player đang aim khi chết
  
FIX: Khi SpectateManager.OnSpectateStarted:
  CameraStateManager.ForceState(CameraState.Free)
  → Unlock camera rotation cho spectator


CAM-2: Spectator camera không có offset/height adjustment
SetCameraTarget(player.transform) → Follow = player root transform
  - Camera sẽ ở vị trí của CinemachineCamera relative to player root
  - Nếu spectated player có camera rig khác (height, offset) → camera lệch
  
FIX: Tìm camera follow point (CameraFollowTarget child) của spectated player
     thay vì dùng root transform


CAM-3: Không có spectator-only camera mode (free-fly)
Hiện tại spectator chỉ có: follow player A → switch → follow player B
Không có free-fly camera (WASD di chuyển camera tự do)

FIX (optional): Thêm InputState.SpectatorFreeCamera
  - WASD di chuyển camera position
  - Mouse rotate
  - Scroll zoom
  - Tab để switch về follow mode


CAM-4: GameCameraController dùng FindFirstObjectByType cho SpectatorInputHandler
Awake(): _spectatorInput = FindFirstObjectByType<SpectatorInputHandler>()
  → Nếu có nhiều SpectatorInputHandler trong scene → lấy sai
  → Nếu SpectatorInputHandler chưa spawn khi Awake() chạy → null
  
FIX: SpectatorInputHandler tự register vào GameCameraController
     hoặc dùng [SerializeField] reference

3. HỆ THỐNG INPUT — PHÂN TÍCH ĐẦY ĐỦ

Kiến trúc hiện tại:
InputLayerManager (Singleton, DefaultExecutionOrder -100)
  - Single Source of Truth cho Enable/Disable ActionMaps
  - Context presets: PlayerAlive, InventoryOpen, Spectating, PlayerDead, ...
  - PushContext/PopContext stack
  - ActionMaps: Player, Combat, Camera, Inventory, Team, UI, Spectator, Objectives, Devices, Debug

InputManager (Singleton, Facade)
  - Expose handlers: MovementHandler, CombatHandler, CameraHandler, ...
  - EnableAllInput() → TransitionToState(PlayerAlive)
  - DisableAllInput() → DisableAll()

Handlers (MonoBehaviour, IInputHandler):
  MovementInputHandler  — Move, Sprint, Crouch, ToggleCameraLock, Jump, Roll
  CombatInputHandler    — Fire, Aim, Reload, WeaponSlot1/2/3
  CameraInputHandler    — Camera rotation, zoom
  InventoryInputHandler — Open/Close inventory, item slots
  InteractionInputHandler — Interact, Hold-interact
  SpectatorInputHandler — NextPlayer, PreviousPlayer, FreeCamera
  UIInputHandler        — UI navigation

Mobile Bridges:
  MobileMovementBridge  — Joystick Pack → MovementInputHandler.SetMobileMove()
  MobilePinchZoomBridge — Pinch gesture → CameraInputHandler zoom
  MobileCameraDragArea  — Drag → camera rotation
  MobileInventoryButton — Mobile inventory toggle
  MobileZoomSlider      — Zoom slider
