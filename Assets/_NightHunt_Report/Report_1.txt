2. PHÂN TÍCH CHI TIẾT TỪNG HỆ THỐNG

🔴 A. NETWORK CONNECTION FLOW — VẤN ĐỀ PHÁT HIỆN

CLIENT FLOW (Ranked_DS):
  match_ready WS → RoomState.DsIp/DsPort set
  ds_ready WS    → NetworkGameManager.NotifyDsReady() + s_dsReadyReceived=true (static)
  SceneLoader.LoadGame() → Unity SceneManager.LoadScene() [KHÔNG phải FishNet]
  
  ⚠ VẤN ĐỀ 1: SceneLoader dùng Unity SceneManager.LoadScene() nhưng
     NetworkGameManager.OnSceneLoadEnd() lắng nghe FishNet SceneManager.OnLoadEnd
     → Fix A đã có (check activeScene.name trong Start()) nhưng là workaround
     → ĐÚNG CHUẨN: nên dùng FishNet SceneManager.LoadGlobalScenes() cho client
     hoặc đồng nhất 1 cách load scene

  ⚠ VẤN ĐỀ 2: s_dsReadyReceived là static field trên NetworkGameManager
     → Nếu player reconnect vào match mới, flag cũ vẫn còn → auto-connect sai
     → Cần reset khi RoomState.ClearRoom() được gọi

  ⚠ VẤN ĐỀ 3: Relay mode (Custom_Relay) dùng SetTransportServerBindAddress("127.0.0.1")
     nhưng không có RelayTransport shim → packet header không được xử lý
     → Relay mode chưa hoàn chỉnh, cần RelayTransport hoặc bỏ hẳn

  ⚠ VẤN ĐỀ 4: Reconnect flow — khi client disconnect giữa chừng:
     - _connectionStarted = true → TryConnectIfReady() bị block
     - TryRetry() chỉ retry 2 lần rồi LoadHome()
     - Không có cơ chế "rejoin" nếu player bị drop nhưng match vẫn đang chạy
     → Cần: sau khi reconnect FishNet, server phải re-spawn player tại vị trí cũ
       hoặc respawn point, không spawn mới hoàn toàn

🔥 PRIORITY 1 — CRITICAL (Blocking production)

P1.1 — Fix Scene Load + Connection Flow
HIỆN TẠI:
  SceneLoader.LoadGame() → Unity SceneManager.LoadScene()
  NetworkGameManager lắng nghe FishNet SceneManager.OnLoadEnd → KHÔNG FIRE

REFACTOR:
  Option A (Recommended): Giữ Unity SceneManager nhưng dùng SceneManager.sceneLoaded event
    trong NetworkGameManager thay vì FishNet OnLoadEnd
    
  Option B: Migrate sang FishNet SceneManager.LoadGlobalScenes() cho client
    (phức tạp hơn, cần test kỹ)

  Fix s_dsReadyReceived reset:
    RoomState.ClearRoom() → NetworkGameManager.ResetConnectionFlags()
    Hoặc: s_dsReadyReceived = false trong LoadHome()
    ✅ Đã có trong LoadHome() nhưng cần verify tất cả exit paths đều gọi LoadHome()


P1.2 — Fix Reconnect Flow
HIỆN TẠI:
  Disconnect → TryRetry (2 lần) → LoadHome()
  Không có "rejoin" — player mất toàn bộ progress

REFACTOR:
  1. RegistryService đã lưu data khi UnregisterPlayer() → tốt
  2. Cần: ServerGameManager.OnPlayerReconnected(conn, backendId)
     - Tìm lại player data từ RegistryService bằng backendId
     - Re-spawn tại respawn point hoặc vị trí cuối
     - Re-assign ownership
  3. Client: sau khi reconnect FishNet, gửi lại ClientNetworkHandler data
     với flag isReconnect = true
  4. Timeout: nếu reconnect không thành công trong 30s → LoadHome()


P1.3 — Fix Score System Unification
HIỆN TẠI:
  ScoringSystem (per-player score) ≠ MatchEndManager (per-team score)
  2 nguồn dữ liệu → inconsistent results

REFACTOR:
  ScoringSystem là SINGLE SOURCE OF TRUTH:
  - AwardKill() → ScoringSystem.AwardKill() → cũng gọi MatchEndManager.AddKill()
  - Hoặc: MatchEndManager đọc từ ScoringSystem thay vì tự track
  - GetFinalResults() dùng ScoringSystem.GetPlayerScore() cho cả player lẫn team
